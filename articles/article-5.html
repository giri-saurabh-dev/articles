Deploying the Lab Hub Dashboard

DATE: 2026.02.17 STATUS: DEPLOYED

The goal was to create a centralized entry point for the local lab environment. This deployment focuses on persistent infrastructure, zero-trust connectivity, and dashboard security.



Infrastructure Note: The server has been pinned to a static IP (10.10.1.36) via Netplan to prevent service disruption during DHCP renewals.

Core Infrastructure

We deployed the Homepage dashboard using Docker. To ensure the dashboard remains accessible under a secure domain, we leveraged a Cloudflare Tunnel, eliminating the need for traditional port forwarding. 

# Docker Environment Configuration
environment:
  - HOMEPAGE_ALLOWED_HOSTS=lab.saurabhgiri.com.np,10.10.1.36

Error Mitigation Log

ISSUE ROOT CAUSE RESOLUTION
DNS_PROBE_FAILED Missing CNAME mapping. Remote Managed Tunnel Route created.
HOST_VERIFICATION Strict security header check. Injected ALLOWED_HOSTS env var.
IP_SHIFT_DISCONNECT DHCP address change (24 -> 36). Manual Netplan static IP override.

Conclusion

The dashboard is now live at lab.saurabhgiri.com.np. The foundation is stable, the tunnel is healthy, and the host verification rules are successfully protecting the dashboard from unauthorized header injection.
